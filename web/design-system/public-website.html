Funeral agencies handle some of the most sensitive personal data imaginable. Sanzu was designed from day one with security, privacy, and regulatory compliance as foundational requirements.

Data Protection
Full GDPR compliance including data minimization, purpose limitation, and right to erasure. All data encrypted at rest (AES-256) and in transit (TLS 1.3). Data residency within the EU guaranteed.

Security practices

How we protect your data every day, at every layer of the platform.

  • Encryption everywhere All data is encrypted at rest using AES-256 and in transit using TLS 1.3. Database-level encryption ensures that even infrastructure access cannot expose raw data.
  • Continuous monitoring Real-time security monitoring with automated alerting for anomalous access patterns, failed authentication attempts, and data exfiltration indicators.
  • Automated backups Daily encrypted backups with point-in-time recovery capability. Backups are stored in geographically separate EU data centers for disaster recovery.
  • Multi-factor authentication All agency accounts support MFA. Admin and manager roles require MFA by default. TOTP and WebAuthn supported.
  • Regular penetration testing Third-party security assessments and penetration tests conducted quarterly. Findings are remediated within SLA timelines and re-verified.
  • Incident response Documented incident response procedures with defined SLAs. Security incidents are communicated to affected tenants within 72 hours as required by GDPR.
